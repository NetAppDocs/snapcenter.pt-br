= Configurar as credenciais para o cluster PostgreSQL
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Você precisa configurar as credenciais de usuário do cluster PostgreSQL para executar operações de proteção de dados.

.Antes de começar
* Se o cluster PostgreSQL não tiver credenciais válidas configuradas, um ícone de cadeado vermelho aparecerá apenas para os recursos descobertos automaticamente. Se, durante uma operação de descoberta subsequente, as credenciais configuradas forem consideradas incorretas ou não fornecerem acesso ao próprio cluster, o ícone de cadeado vermelho reaparecerá.
* Você precisa configurar credenciais válidas do cluster PostgreSQL para poder proteger o recurso ou adicioná-lo a um grupo de recursos para executar operações de proteção de dados.
* O usuário do PostgreSQL configurado nas credenciais deve ter privilégios suficientes para acessar o cluster e executar operações de backup e restauração. No mínimo, o usuário deve ter a função *SUPERUSER* ou privilégios equivalentes (como as permissões *pg_read_all_data*, *pg_write_all_data* e *replication*).
* SnapCenter usa as credenciais configuradas para conectar-se à instância do PostgreSQL via o cliente *psql* ou *libpq*. Certifique-se de que as ferramentas do cliente PostgreSQL estejam instaladas no host do plug-in.



NOTE: Antes de configurar o cluster, você deve primeiro adicionar as credenciais do PostgreSQL ao SnapCenter.

.Passos
. No painel de navegação à esquerda, selecione *Recursos* e, em seguida, selecione SnapCenter Plug-in para PostgreSQL cluster na lista.
. Na página recursos, selecione o tipo de recurso na lista *Exibir*.
. (Opcional) Selecione image:../media/filter_icon.gif["ícone de filtro"] e selecione o nome do host.
+
Em seguida, você pode selecionar image:../media/filter_icon.gif["ícone de filtro"] para fechar o painel de filtros.

. Selecione o cluster e, em seguida, selecione *Configurar credenciais*.
. Selecione a credencial que foi configurada no SnapCenter para o usuário do cluster PostgreSQL. A credencial deve incluir um nome de usuário e senha válidos com privilégios suficientes para executar operações de proteção de dados.
. Selecione *OK*.


Você pode modificar a configuração do cluster na página Topologia.
